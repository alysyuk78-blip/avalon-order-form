# Avalon Order Form — Інструкція з розгортання

## Що в проекті

```
avalon-order-form/
├── public/
│   └── index.html          ← Форма (фронтенд, візуально ідентична Base44)
├── api/
│   └── order.js            ← Серверна функція (Telegram + Google Sheets)
├── google-apps-script.js   ← Скрипт для Google Sheets webhook
├── vercel.json             ← Конфігурація Vercel
├── package.json            ← Залежності
├── .env.example            ← Шаблон змінних оточення
└── README.md               ← Ця інструкція
```

---

## Крок 1: Створити Telegram бота

1. Відкрий Telegram, знайди **@BotFather**
2. Відправ `/newbot`
3. Дай ім'я боту (наприклад: "Avalon Orders Bot")
4. Дай username (наприклад: `avalon_orders_bot`)
5. **Скопіюй токен** — він виглядає як `1234567890:ABCdefGHIjklMNOpqrSTUvwxYZ`
6. **Додай бота в групу** або чат, де хочеш отримувати замовлення
7. Відправ будь-яке повідомлення в цю групу
8. Відкрий у браузері:
   ```
   https://api.telegram.org/bot<ТВІЙ_ТОКЕН>/getUpdates
   ```
9. Знайди `"chat":{"id":-1001234567890}` — це твій **CHAT_ID**

> Якщо chat_id не з'являється — видали бота з групи, додай знову, і знову відправ повідомлення.

## Крок 2: Налаштувати Google Sheets

1. Створи нову **Google Таблицю** (sheets.google.com)
2. Назви її, наприклад: "Avalon — Замовлення"
3. Відкрий **Розширення → Apps Script**
4. Видали дефолтний код і вставте вміст файлу `google-apps-script.js`
5. Натисни **"Розгортання" → "Нове розгортання"**
6. Тип: **"Веб-додаток"**
7. Виконання від: **"Я"** (ваш email)
8. Хто має доступ: **"Будь-хто"**
9. Натисни **"Розгорнути"**
10. **Скопіюй URL** веб-додатку — він виглядає як:
    ```
    https://script.google.com/macros/s/AKfycbx.../exec
    ```

## Крок 3: Розгорнути на Vercel

### Варіант А: Через GitHub (рекомендовано)

1. Створи репозиторій на GitHub
2. Завантаж весь проект `avalon-order-form/` в репозиторій
3. Зайди на [vercel.com](https://vercel.com) → зареєструйся через GitHub
4. Натисни **"Add New Project"** → обери свій репозиторій
5. **Framework Preset**: `Other`
6. Натисни **"Deploy"**

### Варіант Б: Через Vercel CLI

```bash
# Встановити Vercel CLI
npm install -g vercel

# У папці проекту:
cd avalon-order-form
vercel

# Слідуй інструкціям
```

### Налаштувати Environment Variables

В Vercel Dashboard:
1. Відкрий проект → **Settings** → **Environment Variables**
2. Додай змінні:

| Name | Value |
|------|-------|
| `TELEGRAM_BOT_TOKEN` | Токен з Кроку 1 |
| `TELEGRAM_CHAT_ID` | Chat ID з Кроку 1 |
| `GOOGLE_SHEET_URL` | URL з Кроку 2 |

3. Натисни **Save**
4. Перезапусти деплоймент: **Deployments** → крайній → **Redeploy**

## Кастомізація

### Зміна даних форми

Відкрий `public/index.html` → знайди об'єкт `SETTINGS` → змінюй:
- Кольори, візерунки, способи оплати, транспортні компанії
- Текст заголовку і опису
- Номер телефону
- URL зображень

### Власний домен

В Vercel Dashboard → **Settings** → **Domains** → додай свій домен.
Наприклад: `order.avalon.ua`

### Зміна зображень

Зображення (логотип, візерунки, схеми розмірів) зараз завантажуються з Base44/Supabase CDN. Рекомендую:
1. Завантажити їх локально в `public/images/`
2. Замінити URL в `SETTINGS` на `/images/filename.png`

Це зробить форму повністю незалежною від Base44.

## Альтернативний варіант: без серверної функції

Якщо не хочеш використовувати Vercel serverless, можна відправляти дані напряму з фронтенду:

1. В `public/index.html` знайди об'єкт `CONFIG`
2. Розкоментуй та заповни:
```javascript
const CONFIG = {
  API_URL: null, // вимкнути серверну функцію
  TELEGRAM_BOT_TOKEN: "YOUR_BOT_TOKEN",
  TELEGRAM_CHAT_ID: "YOUR_CHAT_ID",
  GOOGLE_SHEET_URL: "YOUR_URL",
};
```
3. Тоді файл `index.html` можна хостити будь-де — навіть на GitHub Pages

> ⚠️ Мінус: токен Telegram-бота буде видно у коді сторінки. Для невеликого бізнесу це прийнятно, але не ідеально.

## FAQ

**Q: Чи потрібно платити за Vercel?**
A: Ні. Безкоштовний план — 100 ГБ трафіку/місяць і 100 000 викликів serverless функцій. Для форми замовлень цього більш ніж достатньо.

**Q: Що буде з файлами візерунків?**
A: Серверна функція відправляє їх як документ прямо в Telegram-чат. Окремий файловий хостинг не потрібен.

**Q: Чи можна підключити Trello пізніше?**
A: Так. В `api/order.js` додай блок аналогічний Telegram з використанням Trello API.

**Q: Чи залишиться зв'язок з Base44?**
A: Зображення поки що завантажуються з Base44 CDN. Скопіюй їх локально для повної незалежності (див. "Зміна зображень" вище).
